다국어 정적 사이트 생성기 만들기

정적 사이트 생성기는 빠르고 안전한 웹사이트를 구축하는데 점점 더 인기를 얻고 있습니다. 이 포스트에서는 다국어 지원 커스텀 SSG를 만든 방법을 공유하겠습니다.

왜 직접 만들까?

Jekyll, Hugo, Gatsby 같은 훌륭한 정적 사이트 생성기들이 많지만, 직접 만들면:

  • 빌드 프로세스를 완전히 제어할 수 있음
  • SSG 작동 원리를 깊이 이해할 수 있음
  • 특정 요구사항에 맞춤화 가능
  • 학습 기회

2. 다국어 지원

생성기는 다음과 같은 다국어 기능을 지원합니다:

  • 언어별 콘텐츠 디렉토리
  • 자동 URL 라우팅
  • 번역 헬퍼
  • 언어 전환기

3. 템플릿 엔진

간단하지만 강력한 템플릿 엔진 구현:

  • 변수 보간
  • 조건문
  • 반복문
  • 부분 템플릿

아키텍처

src/
├── generator/     # 코어 빌드 로직
├── templates/     # HTML 템플릿
├── utils/        # 헬퍼 함수
└── i18n/         # 번역

content/
├── posts/
│   ├── en/      # 영어 포스트
│   └── ko/      # 한국어 포스트
└── pages/
    ├── en/      # 영어 페이지
    └── ko/      # 한국어 페이지

성능 최적화

  • 빠른 빌드를 위한 병렬 처리
  • 템플릿 캐싱
  • 점진적 빌드 (개발 중)
  • 자산 최적화

도전과 해결책

  1. URL 구조: 유연한 라우팅 시스템 구현
  2. 자산 관리: 효율성을 위한 중앙화된 자산
  3. 빌드 속도: 병렬화와 캐싱 추가
