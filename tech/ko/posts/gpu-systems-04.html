лӢӨмқҢм—җлҠ” мқҙ нқҗлҰ„мқ„ ліҙм„ёмҡ”

PyTorch лӮҙл¶Җ кө¬мЎ° н…җм„ң, autograd, CUDA extensionмқ„ мқҙн•ҙн•ҳкі  м»ӨмҠӨн…Җ м»Өл„җмқ„ мӢӨм ң н•ҷмҠө мҪ”л“ңм—җ м—°кІ°н•ҳкё° л¶„мӮ° LLM н•ҷмҠө data parallelл¶Җн„° tensor parallel, FSDP, ZeRO, к·ёлҰ¬кі  нҳ„лҢҖ LLM н•ҷмҠө н”„л Ҳмһ„мӣҢнҒ¬к№Ңм§Җ м»ҙн“Ён„° кө¬мЎ° CPU лӮҙл¶Җ кө¬мЎ°мҷҖ к¶Ңн•ң мҲҳмӨҖл¶Җн„° л©”лӘЁлҰ¬ кі„мёөкіј нҳ„лҢҖ л©ҖнӢ°мҪ”м–ҙ н”„лЎңм„ём„ңк№Ңм§Җ

м»Өл„җмқ„ м§Ғм ‘ м“°кё° мӢңмһ‘н•ҳл©ҙ мғқкё°лҠ” м§Ҳл¬ё

GPU м•„нӮӨн…ҚмІҳлҘј м–ҙлҠҗ м •лҸ„ мқҙн•ҙн–ҲлӢӨл©ҙ мқҙм ңлҠ” м§Ғм ‘ CUDA kernelмқ„ мҚЁлҙҗм•ј н•ңлӢӨ. к·ёлҹ°лҚ° л§үмғҒ мӢңмһ‘н•ҳл©ҙ л¬ёлІ•ліҙлӢӨ лҚ” н—·к°ҲлҰ¬лҠ” л¶Җл¶„мқҙ мһҲлӢӨ. л°”лЎң thread indexingкіј launch configurationмқҙлӢӨ.

мҳҲлҘј л“Өм–ҙ мқҙлҹ° м§Ҳл¬ёмқҙ л°”лЎң лӮҳмҳЁлӢӨ.

  • blockмқ„ 128лЎң мһЎм•„м•ј н•ҳлӮҳ, 256мңјлЎң мһЎм•„м•ј н•ҳлӮҳ?
  • 1D gridлЎң м¶©л¶„н•ңк°Җ, 2Dк°Җ лҚ” лӮҳмқҖк°Җ?
  • thread н•ҳлӮҳк°Җ мӣҗмҶҢ н•ҳлӮҳл§Ң мІҳлҰ¬н•ҙм•ј н•ҳлӮҳ?

мқҙ м§Ҳл¬ёл“ӨмқҖ м •лӢө н•ҳлӮҳк°Җ мһҲлҠ” л¬ём ңк°Җ м•„лӢҲлқј, л¬ём ң кө¬мЎ°мҷҖ н•ҳл“ңмӣЁм–ҙ м ңм•Ҫмқ„ к°ҷмқҙ лҙҗм•ј н•ҳлҠ” л¬ём ңлӢӨ.

к°ҖмһҘ кё°ліём Ғмқё indexing

CUDA kernelм—җм„ңлҠ” ліҙнҶө blockIdx, blockDim, threadIdxлҘј мЎ°н•©н•ҙ мһҗмӢ мқҳ мһ‘м—… мң„м№ҳлҘј кі„мӮ°н•ңлӢӨ.

vector addмІҳлҹј 1м°Ёмӣҗ лҚ°мқҙн„°лқјл©ҙ лӢӨмқҢ mental modelмқҙл©ҙ м¶©л¶„н•ҳлӢӨ.

  • blockмқҖ нҒ° кө¬к°„мқ„ лӮҳлҲҲлӢӨ
  • block м•Ҳ threadлҠ” к·ё кө¬к°„ м•Ҳмқҳ м„ёл¶Җ мӣҗмҶҢлҘј лӢҙлӢ№н•ңлӢӨ
  • м „мІҙ indexлҠ” blockIdx.x * blockDim.x + threadIdx.xлЎң кі„мӮ°н•ңлӢӨ

мқҙ мһҗмІҙлҠ” лӢЁмҲңн•ҳлӢӨ. н•ҳм§Җл§Ң м§„м§ң мӨ‘мҡ”н•ң кІғмқҖ launch configurationмқ„ нҶөн•ҙ м „мІҙ мһ‘м—…мқ„ м–ҙл–»кІҢ л¶„л°°н•ҳлҠҗлғҗмқҙлӢӨ.

м—¬кё°м„ң л§ҺмқҖ мҙҲмӢ¬мһҗк°Җ indexingмқ„ лӢЁмҲң boilerplateмІҳлҹј ліёлӢӨ. н•ҳм§Җл§Ң мӢӨм ңлЎңлҠ” л¬ём ңлҘј threadм—җ м–ҙл–»кІҢ мӮ¬мғҒн•  кІғмқёк°ҖлҘј кІ°м •н•ҳлҠ” н•өмӢ¬ лӢЁкі„лӢӨ. кІ°көӯ к°ҷмқҖ м—°мӮ°лҸ„ indexingкіј launch shapeм—җ л”°лқј memory patternкіј нҳ‘м—… кө¬мЎ°к°Җ нҒ¬кІҢ лӢ¬лқјм§„лӢӨ.

block sizeлҠ” мҷң мӨ‘мҡ”н•ңк°Җ

block sizeлҠ” лӢЁмҲңнһҲ thread мҲҳлҘј м •н•ҳлҠ” к°’мқҙ м•„лӢҲлӢӨ. мӢӨм ңлЎңлҠ” лӢӨмқҢкіј к°ҷмқҙ м—¬лҹ¬ мҡ”мҶҢм—җ мҳҒн–Ҙмқ„ мӨҖлӢӨ.

  • warp мҲҳ
  • shared memory мӮ¬мҡ© нҢЁн„ҙ
  • register pressure
  • occupancy

мҳҲлҘј л“Өм–ҙ block sizeлҘј 32мқҳ л°°мҲҳлЎң мһЎлҠ” кІғмқҖ кұ°мқҳ кё°ліёмқҙлӢӨ. warpк°Җ 32 threadмқҙкё° л•Ңл¬ёмқҙлӢӨ. к·ёл ҮлӢӨкі  н•ӯмғҒ нҒ¬кІҢ мһЎлҠ” кІғмқҙ мўӢмқҖ кІғмқҖ м•„лӢҲлӢӨ. л„Ҳл¬ҙ нҒ° blockмқҖ shared memoryмҷҖ registerлҘј л§Һмқҙ м җмң н•ҙ лҸҷмӢңм—җ мҳ¬лқјк°ҖлҠ” block мҲҳлҘј мӨ„мқј мҲҳ мһҲлӢӨ.

мӢӨл¬ҙм—җм„ңлҠ” 128, 256, 512 к°ҷмқҖ к°’мңјлЎң мӢңмһ‘н•ҙм„ң н”„лЎңнҢҢмқјл§ҒмңјлЎң л№„көҗн•ҳлҠ” кІҪмҡ°к°Җ л§ҺлӢӨ. мӨ‘мҡ”н•ң кІғмқҖ мҲ«мһҗлҘј мҷёмҡ°лҠ” кІғмқҙ м•„лӢҲлқј, мҷң к·ё л№„көҗк°Җ н•„мҡ”н•ңм§Җ мқҙн•ҙн•ҳлҠ” кІғмқҙлӢӨ.

мҰү, block sizeлҠ” н•ҳл“ңмӣЁм–ҙк°Җ мўӢм•„н•ҳлҠ” мҲ«мһҗлҘј м•”кё°н•ҳлҠ” л¬ём ңк°Җ м•„лӢҲлқј, лӮҙ м»Өл„җмқҳ shared memory мӮ¬мҡ©, register pressure, warp мҲҳмҷҖ м–ҙл–Ө к· нҳ•мқ„ мқҙлЈЁлҠ”м§Җ ліҙлҠ” л¬ём ңлӢӨ.

grid нҒ¬кё°лҠ” GPUлҘј м–јл§ҲлӮҳ мұ„мҡ°лҠ”к°ҖмҷҖ м—°кІ°лҗңлӢӨ

block sizeк°Җ block лӮҙл¶Җ кө¬мЎ°лқјл©ҙ, grid sizeлҠ” м „мІҙ GPUлҘј м–јл§ҲлӮҳ л°”мҒҳкІҢ л§Ңл“Ө мҲҳ мһҲлҠ”м§ҖмҷҖ м—°кІ°лҗңлӢӨ.

л¬ём ңк°Җ л„Ҳл¬ҙ мһ‘мңјл©ҙ GPU м „мІҙлҘј м¶©л¶„нһҲ нҷңмҡ©н•ҳм§Җ лӘ»н•ңлӢӨ. л°ҳлҢҖлЎң л¬ём ңк°Җ м¶©л¶„нһҲ нҒ¬л©ҙ GPUк°Җ л°”мҒҳкІҢ лҸҢм•„к°Җм§Җл§Ң, к·ё м•Ҳм—җм„ңлҸ„ memory-boundмқём§Җ compute-boundмқём§Җм—җ л”°лқј мӢӨм ң м„ұлҠҘмқҖ лӢ¬лқјм§„лӢӨ.

мҰү, grid sizeлҠ” "лӘЁл“  лҚ°мқҙн„° мӣҗмҶҢлҘј лҚ®лҠ”к°Җ" мҲҳмӨҖм—җм„ң лҒқлӮҳлҠ” кІғмқҙ м•„лӢҲлқј, GPUк°Җ лҶҖм§Җ м•ҠлҸ„лЎқ м¶©л¶„н•ң лі‘л ¬м„ұмқ„ м ңкіөн•ҳлҠ”к°Җк№Ңм§Җ лҙҗм•ј н•ңлӢӨ.

thread н•ҳлӮҳк°Җ мӣҗмҶҢ н•ҳлӮҳл§Ң мІҳлҰ¬н•ҙм•ј н• к№Ң?

мҙҲкё° мҳҲм ңм—җм„ңлҠ” ліҙнҶө thread н•ҳлӮҳк°Җ мӣҗмҶҢ н•ҳлӮҳлҘј л§ЎлҠ”лӢӨ. н•ҳм§Җл§Ң мӢӨм ң мөңм Ғнҷ”м—җм„ңлҠ” thread н•ҳлӮҳк°Җ м—¬лҹ¬ мӣҗмҶҢлҘј лӢӨлЈЁлҠ” кІғмқҙ лҚ” лӮҳмқ„ л•ҢлҸ„ л§ҺлӢӨ.

мқҙмң лҠ” лӢӨмқҢкіј к°ҷлӢӨ.

  • л©”лӘЁлҰ¬ м ‘к·ј нҢЁн„ҙмқ„ лҚ” мһҳ л§һм¶ң мҲҳ мһҲлӢӨ
  • register мһ¬мӮ¬мҡ©мқҙ к°ҖлҠҘн•ҳлӢӨ
  • loop unrollingкіј кІ°н•©н•ҳкё° мўӢлӢӨ

л¬јлЎ  к·ёл§ҢнҒј register мӮ¬мҡ©мқҙ лҠҳкі  мҪ”л“ңк°Җ ліөмһЎн•ҙм§Ҳ мҲҳ мһҲлӢӨ. к·ёлһҳм„ң м—¬кё°м„ңлҸ„ н•өмӢ¬мқҖ к· нҳ•мқҙлӢӨ.

мӢӨм ңлЎңлҠ” мқҙ м§Ҳл¬ёмқҙ мқҙнӣ„ vectorized load/store, register blocking, tile design к°ҷмқҖ лҚ” кі кёү мөңм Ғнҷ” мЈјм ңлЎң мқҙм–ҙм§„лӢӨ. кІ°көӯ launch configurationмқҖ мҙҲл°ҳ м„Өм •к°’мқҙ м•„лӢҲлқј, м»Өл„җ м„ұлҠҘ м„Өкі„ м „мІҙмқҳ м¶ңл°ңм җмқҙлӢӨ.

launch configurationмқ„ м •н•  л•Ң ліҙлҠ” кё°мӨҖ

м»Өл„җмқ„ мһ‘м„ұн•  л•ҢлҠ” мөңмҶҢн•ң м•„лһҳ кё°мӨҖмқ„ н•Ёк»ҳ ліҙлҠ” нҺёмқҙ мўӢлӢӨ.

  • warp лӢЁмң„м—җ л§һлҠ” block sizeмқёк°Җ?
  • shared memory мӮ¬мҡ©лҹүмқҖ м Ғм Ҳн•ңк°Җ?
  • register мӮ¬мҡ©мқҙ occupancyлҘј м§ҖлӮҳм№ҳкІҢ к№Һм§Җ м•ҠлҠ”к°Җ?
  • лҚ°мқҙн„° лӘЁм–‘м—җ л§һлҠ” 1D/2D/3D gridмқёк°Җ?
  • thread н•ҳлӮҳк°Җ мІҳлҰ¬н•ҳлҠ” лӢЁмң„к°Җ л„Ҳл¬ҙ мһ‘кұ°лӮҳ нҒ¬м§Җ м•ҠмқҖк°Җ?

мқҙ кё°мӨҖмқҙ мһЎнҳҖ мһҲмңјл©ҙ kernel launchлҠ” лӢЁмҲңн•ң boilerplateк°Җ м•„лӢҲлқј м„ұлҠҘ м„Өкі„мқҳ мқјл¶ҖлЎң ліҙмқҙкё° мӢңмһ‘н•ңлӢӨ.

мҳҲмӢңлЎң matrix multiplyлҘј ліҙл©ҙ

matrix multiplyлҠ” ліҙнҶө 2D indexingмқ„ м“ҙлӢӨ. мқҙмң лҠ” м¶ңл Ҙ н–үл ¬мқҳ (row, col) мң„м№ҳлҘј threadм—җ мһҗм—°мҠӨлҹҪкІҢ л§Өн•‘н•  мҲҳ мһҲкё° л•Ңл¬ёмқҙлӢӨ.

мқҙл•Ң naiveн•ҳкІҢлҠ” thread н•ҳлӮҳк°Җ м¶ңл Ҙ мӣҗмҶҢ н•ҳлӮҳлҘј кі„мӮ°н•ҳкІҢ н•  мҲҳ мһҲлӢӨ. н•ҳм§Җл§Ң мЎ°кёҲ лҚ” лӮҳм•„к°Җл©ҙ blockмқҙ м¶ңл Ҙ нғҖмқј н•ҳлӮҳлҘј л§Ўкі , threadл“Өмқҙ к·ё м•Ҳм—җм„ң нҳ‘м—…н•ҳлҸ„лЎқ л°”лҖҗлӢӨ. кІ°көӯ launch configurationмқҙ кі§ нғҖмқј м „лһөкіј м—°кІ°лҗңлӢӨ.

м •лҰ¬

CUDA kernel мһ‘м„ұм—җм„ң мӨ‘мҡ”н•ң кІғмқҖ л¬ёлІ•ліҙлӢӨ л§Өн•‘мқҙлӢӨ.

  • л¬ём ңлҘј threadм—җ м–ҙл–»кІҢ лӮҳлҲҢ кІғмқёк°Җ
  • threadлҘј warpмҷҖ blockмңјлЎң м–ҙл–»кІҢ л¬¶мқ„ кІғмқёк°Җ
  • к·ё м„ нғқмқҙ л©”лӘЁлҰ¬мҷҖ occupancyм—җ м–ҙл–Ө мҳҒн–Ҙмқ„ мЈјлҠ”к°Җ

мқҙ к°җк°Ғмқҙ мғқкё°л©ҙ kernel launch м„Өм •мқҙ лҚ” мқҙмғҒ мҷёмӣҢм•ј н•ҳлҠ” мҲ«мһҗк°Җ м•„лӢҲлқј, м„ұлҠҘмқ„ м„Өкі„н•ҳлҠ” мҶҗмһЎмқҙмІҳлҹј ліҙмқҙкІҢ лҗңлӢӨ.

лӢӨмқҢ кёҖм—җм„ңлҠ” memory coalescing, shared memory, reduction нҢЁн„ҙмқ„ мӨ‘мӢ¬мңјлЎң лҚ” мӢӨм ңм Ғмқё м»Өл„җ мөңм Ғнҷ” нҸ¬мқёнҠёлҘј ліёлӢӨ.

мқҙм „ кёҖ мҳӨнҶ л§ҲнғҖмҷҖ м»ҙнҢҢмқјлҹ¬ 08 - мғҒн–ҘмӢқ нҢҢмӢұ
