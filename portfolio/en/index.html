Hardcoded Rules → Plugin Architecture

Before

Each type had its own copy-pasted decision logic. Adding a new type meant forking the codebase. Changing a policy meant touching the entire execution flow.

After

A 3-layer plugin architecture separating state, decision logic, and execution. New types added by composing layers. 10+ types on one framework.

  • State layer — Abstract interface defines shared contracts, type-specific differences via composition. 9 variants in production
  • Planning layer — Manages decision lifecycle. 7 concrete planners (~3,230 LOC) inherit from a base ABC
  • Execution layer — Template method pattern. Type-specific executors dynamically registered at runtime

Greedy Allocation → Multi-Resource Transactions

Before

"Assign work to whatever is free right now." No atomic allocation across resources. Failure midway left state corrupted with no rollback.

After

Entire paths reserved as a single transaction. Failure at any point triggers automatic rollback.

  • Coordinator + 3 type-specific reservers handling constraints independently
  • Path explorer ranking routes by state transition costs
  • trial/rollback mechanism with automatic rollback on failure + dedup cache

God Object → Explicit Ownership + Event-Driven Architecture

Before

3,800 lines, 114 attributes, 6 tangled concerns. __getattr__ proxies hiding ownership. Callback chains making side effects untraceable.

After

226-line core + 19 components + 25 facades. Zero proxies. Pub/sub event bus (29 LOC) fully decoupling execution from side effects.

  • 114 attributes → 19 components + 25 facades. Removed proxies from 152+ files
  • Pub/sub event bus (29 LOC) fully decoupling execution from side effects
  • -752 / +163 lines. New features no longer require touching existing flows

Slow Hot Paths + Black Box → Optimization + Observability

Before

Hot paths called 22M–500M times per run. No way to trace why a decision was made.

After

Profiling-driven hot path optimization. Full decision traceability from outcome back to cause.

  • Enum caching (22M+), dict indexes, local caches, custom __deepcopy__ for hot path optimization
  • Gantt chart + event timeline tracing decisions back to root cause
  • Structured logging with 105-module enum taxonomy
