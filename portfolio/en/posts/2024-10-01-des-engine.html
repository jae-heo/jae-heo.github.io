High-Performance Discrete Event Simulation Engine

Overview

Developed a Python-based discrete event simulation engine capable of handling millions of entities with sub-millisecond event processing latency. This engine powers large-scale simulations for network modeling, logistics optimization, and agent-based modeling.

Technical Highlights

Performance Achievements

  • 10 million events/second throughput on single node
  • Sub-millisecond event scheduling with Calendar Queue
  • 100x speedup through C extensions and JIT compilation
  • Linear scalability up to 64 cores with distributed processing

Core Technologies

  • Python 3.11+ with type hints and async/await
  • Cython for performance-critical paths
  • NumPy for vectorized operations
  • Ray for distributed simulation
  • MPI for HPC cluster deployment

Key Features

Adaptive Scheduling

Implemented dynamic scheduler that automatically switches between Calendar Queue, Ladder Queue, and Splay Tree based on workload characteristics, achieving O(1) average complexity.

Zero-Copy Architecture

Designed shared memory system using memory-mapped files and buffer protocol, eliminating data copying overhead in multi-process simulations.

Time Warp Protocol

Implemented Jefferson's Virtual Time with optimistic synchronization, enabling efficient distributed simulation with automatic rollback and anti-message handling.

Impact

  • Reduced simulation runtime from days to hours for million-entity models
  • Enabled real-time decision support for logistics networks
  • Open-sourced core components with 500+ GitHub stars
  • Adopted by 3 research institutions for agent-based modeling

Code Sample

from simulation import Engine, CalendarQueue

engine = Engine(scheduler=CalendarQueue())
engine.schedule(time=0.0, event=InitializeSimulation())
engine.run(until=86400.0)  # Run for 24 hours simulated time
